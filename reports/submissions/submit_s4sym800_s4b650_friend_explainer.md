# submit_s4sym800_s4b650.csv 설명

## 한 줄 요약

`submit_s4sym800_s4b650.csv`는 ETRI 대회 제출용 CSV 파일입니다.  
기존 LightGBM 기반 예측 결과를 바탕으로, 7개 타깃 중 **S4 예측값만 추가 조정**해서 만든 후처리 제출 파일입니다.

## 파일의 역할

이 CSV는 대회 플랫폼에 업로드하는 최종 제출 파일입니다.  
각 행은 테스트 데이터의 하루 기록 하나를 의미하고, 각 타깃 컬럼에는 해당 타깃이 `1`일 확률이 들어 있습니다.

파일 구조는 샘플 제출 파일과 동일합니다.

| 컬럼 | 의미 |
|---|---|
| `subject_id` | 사용자 ID |
| `sleep_date` | 수면 기준 날짜 |
| `lifelog_date` | 라이프로그 기준 날짜 |
| `Q1` | Q1 타깃의 예측 확률 |
| `Q2` | Q2 타깃의 예측 확률 |
| `Q3` | Q3 타깃의 예측 확률 |
| `S1` | S1 타깃의 예측 확률 |
| `S2` | S2 타깃의 예측 확률 |
| `S3` | S3 타깃의 예측 확률 |
| `S4` | S4 타깃의 예측 확률 |

## 사용한 기본 예측

이 파일은 완전히 새로 학습한 모델의 결과가 아닙니다.  
기본이 되는 예측은 기존 LightGBM 계열의 temporal target-wise 예측입니다.

핵심 기준 파일은 다음입니다.

```text
lgb_temporal_s4b650.csv
```

`submit_s4sym800_s4b650.csv`는 이 기준 파일에서 대부분의 타깃은 그대로 유지하고, S4만 조금 더 조정한 버전입니다.

## 어떤 부분을 바꿨나?

7개 타깃 중 아래 6개는 기준 파일의 예측값을 그대로 사용했습니다.

```text
Q1, Q2, Q3, S1, S2, S3
```

변경한 타깃은 하나입니다.

```text
S4
```

즉 이 CSV는 전체 모델을 바꾼 파일이라기보다, **S4만 따로 실험하기 위해 만든 S4 중심 후처리 파일**입니다.

## S4 조정 방식

S4는 두 예측 축 사이의 차이를 이용해 조정했습니다.

사용한 기준은 다음 두 예측입니다.

```text
public_lgb_targetwise_temporal_targetwise_v1
public_lgb_targetwise_temporal_targetwise_s4_v1
```

간단히 말하면:

- 첫 번째 예측은 S4를 많이 움직이기 전의 기준 예측
- 두 번째 예측은 S4에 temporal 정보를 반영한 예측
- 두 예측의 차이를 이용해서 S4 값을 추가로 이동

계산 아이디어는 아래와 같습니다.

```text
S4_new = S4_anchor + beta * (S4_temporal - S4_anchor)
```

여기서 `submit_s4sym800_s4b650.csv`는 beta 값을 `8.00`으로 둔 파일입니다.

## 파일 이름의 의미

파일명 `submit_s4sym800_s4b650.csv`는 다음 의미를 가집니다.

| 부분 | 의미 |
|---|---|
| `submit` | 대회 제출용 파일 |
| `s4` | S4 타깃을 중심으로 조정 |
| `sym800` | symmetric 방식으로 beta `8.00` 적용 |
| `s4b650` | 기존 기준이 beta `6.50` S4 파일이라는 의미 |

여기서 `sym`은 S4 값이 올라가는 행과 내려가는 행을 따로 나누지 않고, 전체 S4 변화 방향을 그대로 적용했다는 뜻입니다.

## 값의 범위 처리

예측 확률은 너무 극단적인 값이 되지 않도록 범위를 제한했습니다.

```text
최소값: 0.02
최대값: 0.98
```

이 처리는 logloss 평가에서 확률이 0이나 1에 너무 가까워지는 위험을 줄이기 위한 것입니다.

## 제출 포맷 검증

이 CSV는 샘플 제출 파일과 같은 형식을 유지하도록 만들었습니다.

검증한 항목은 다음입니다.

- 행 개수: 250개 테스트 행
- 컬럼 개수: 10개
- key 컬럼 순서 유지
- `subject_id`, `sleep_date`, `lifelog_date` 원본 샘플과 일치
- 예측값은 모두 숫자
- 예측값은 모두 유효한 확률 범위 안에 있음

## 이 파일의 목적

이 파일의 목적은 전체 모델을 바꾸는 것이 아니라, **S4 타깃만 따로 강하게 조정했을 때 제출 결과가 어떻게 달라지는지 확인하는 것**입니다.

따라서 이 CSV는 다음 성격을 가집니다.

- LightGBM 기반 예측을 사용
- 새 모델 학습 파일이 아니라 후처리 제출 파일
- Q1/Q2/Q3/S1/S2/S3는 고정
- S4만 beta `8.00`으로 조정
- 샘플 제출 형식에 맞춘 대회 업로드용 CSV
